# 代码变更审计报告

## 1. 基本信息

- 变更编号：CA-M2-RX-TX-FORMAT-001
- 标题：收发格式改为可选择（HEX / 文本）
- 基线版本：V1.2.5
- 审计规范版本：V1.2
- 里程碑：M2
- 日期：2026-07-21
- 范围：CommunicationAssistant 主窗收发显示/发送格式控件

## 2. 变更目标

- 原问题：接收仅「十六进制显示」勾选，发送仅「十六进制发送」勾选，不像串口助手那样显式可选。
- 预期：接收显示、Native 发送均为下拉选择 `HEX` / `文本`；Legacy 仍用原有 Legacy 模式框。
- 明确不做：不改 DLL；不改列表「格式」列；不新增示例中的断帧/脚本等能力。

## 3. 实际修改

| 文件 | 动作 |
|---|---|
| `ui/MainWindow.ui` | 侧栏改为「接收设置」+ `rxFormatCombo`；单条发送区 `txFormatCombo` |
| `src/MainWindow.h/.cpp` | 成员替换；`preferHexDisplay` / `preferHexSend`；同步可见性 |

## 4. 行为说明

| 控件 | 选项 | 作用 |
|---|---|---|
| 接收显示 | HEX / 文本 | 数据窗载荷展示格式 |

| Legacy 模式框 | 自动/数值/文本 | 仅 Legacy 可见；选 HEX 仍拒绝并写边界日志 |
