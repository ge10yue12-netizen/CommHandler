# 通信协议开发完整手册（基于 Demo_test + CommHandler）

> **目的**：让你能独立做出「试验机 ↔ 软件」完整通信：协议增删改查、组包发送、收包解包、正确性校验、联调排错，并学会加一个极简协议。  
> **工程对照**：  
> - 软件端：`CommLab`（调 `CommHandler.dll`）  
> - 试验机端：`MachinePeer`（纯 Qt，不调 DLL）  
> - 库源码：`CommHandler`（`SocketComm` / `SerialPortComm`）  
> **原则**：跟库设计；库没有的路径不要假装实现；指令 ≠ 测数。  
> **联调步骤表**：见同目录 `测试手册.md`。

## 1. 总体架构与数据流

### 1.1 两端角色

| 端 | 工程 | 是否链 DLL | 职责 |
|----|------|------------|------|
| 软件 | CommLab | 是 | `setParameter` → `Connect` → 收库信号 → 处理 → `SendData(vector)` |
| 试验机 | MachinePeer | 否 | 按库**入站格式**组包发送；按库**出站格式**解软件回包 |

同一时刻 **只开串口或只开网口**（互斥）。

### 1.2 业务闭环（正确模型）

```
试验机 ──指令──► 库解析 ──emitEventMsg──► 软件（同步运行状态，不回发力温）
试验机 ──测数──► 库解析 ──emitNewData───► 软件处理 ──SendData(vector)──► 库组出站帧 ──► 试验机解包使用
```

| 注意 | 说明 |
|------|------|
| 库自动 ACK | 如 JSON 对 `tn=1` 的应答 ≠ 业务力温回发 |
| 指令不连带测数 | 「开始」只发指令帧 |
| 无入站则无闭环 | 如串口冠腾：库有出站 `R#N#`，无入站测数解析 → Demo 禁「发力温」 |

### 1.3 本仓库关键文件（对照学习）

| 文件 | 作用 |
|------|------|
| `CommLab/src/DemoComm.h` | 软件：`setupSerial` / `setupNetwork` / `replyProcessed` |
| `CommLab/src/ProtoCapability.h` | 软件：按库能力门控 |
| `CommLab/src/CommController.*` | 线程安全转发 `emitNewData` / `emitEventMsg` |
| `CommLab/src/MainWindow.*` | 软件 UI 与业务编排 |
| `MachinePeer/src/DemoComm.h` | 试验机：组库入站帧 |
| `MachinePeer/src/ReplyParse.h` | 试验机：解库出站帧 |
| `MachinePeer/src/ProtoCapability.h` | 试验机：按钮/组包门控 |
| 库 `SocketComm.cpp` | 网口收发 `switch(iProtoType)` |
| 库 `SerialPortComm.cpp` | 串口收发 `switch(iProtocolType)` |

### 1.4 能力速查（审计结论，写代码前先看）

**网口 `iProtoType`**

| PT | 收指令 | 收测数 | `SendData(vector)` |
|----|--------|--------|-------------------|
| 0 JSON | ✓ | ✗ | ✓（出站 `tn=2`） |
| 1 万测 | ✓ | ✗ | ✓ |
| 2 中机 | ✓ | ✓ | ✓ |
| 3 三思 | ✗ | ✓ | ✗ |
| 4 触发存图 | 存图 | ✗ | ✗ |
| 5 威盛 | ✗ | ✓ | ✗ |
| 6/7 纳百川 | ✓ | ✗ | ✓ |

**串口 `iProtocolType`**

| PT | 收指令 | 收测数 | `SendData(vector)` |
|----|--------|--------|-------------------|
| 0 三思 | ✓ | ✓ | ✓ |
| 1 科新 | ✗ | ✓ | ✓（**仅力**） |
| 2 时代新材 | ✗ | ✓ | ✗（空实现） |
| 3 IEEE | ✓ | ✗ | ✓ |
| 4 冠腾 | ✗ | ✗ | ✓（`R#N#`） |

## 2. 协议的增删改查（怎么用）

协议分两层：**库协议**（真正收发）与 **Demo 门控/组包**（跟库对齐）。

### 2.1 查（Query）——怎么知道有哪些协议、能不能用

1. 看库 `switch`：  
   - 网口：`SocketComm::onRecvDataSlots` / `SendData` 的 `iProtoType`  
   - 串口：`SerialPortComm::onRecvDataSlots` / `SendData` 的 `iProtocolType`  
2. 看 Demo：`ProtoGuide` 下拉 + `ProtoCapability` 门控。  
3. **查的结果要回答四件事**：收指令？收测数？业务回发？帧格式？

示例（查串口三思）：

```text
iProtocolType = 0
收指令：3D0D/3E0D/6B0D/5A0D → emitEventMsg
收测数：E2 + 4 字节力原始量 → emitNewData
回发：FloatToHex(...) + 0x0D
```

### 2.2 增（Create）——新增一种协议要改哪里

| 层级 | 必须改 | 说明 |
|------|--------|------|
| **库（真正生效）** | `onRecv*` 增加 `case N` | 解入站 → `emitNewData` / `emitEventMsg` |
| **库** | `SendData` 增加 `case N` | `vector` → 出站字节 |
| **库** | 默认值/文档/枚举注释 | 避免索引错位 |
| **Demo** | `ProtoGuide` 下拉一项 | `itemData = N` |
| **Demo** | `ProtoCapability` | 四个能力 bool |
| **Demo** | `DemoComm` 组包 | 试验机发什么 |
| **Demo** | `ReplyParse` 解包 | 试验机解软件回什么 |

> **只改 Demo、不改库**：试验机可以发出任意字节，但软件侧库不会解析 → **假闭环**。冠腾即如此：出站有代码、入站无 case。

### 2.3 改（Update）——改已有协议

1. 先改库收/发格式（唯一真相）。  
2. 同步 Peer `pack*` / `ReplyParse`。  
3. 同步手册与 `ProtoCapability`。  
4. 回归：指令用例 + 测数用例 + 回发用例。

### 2.4 删（Delete）——废弃协议

1. 库 `case` 可保留但文档标废弃，或 `default` 丢弃。  
2. Demo 下拉移除或灰掉并写拒绝原因。  
3. **不要**删一半：下拉还在但组包还指向旧格式 → 联调必乱。

### 2.5 软件侧「选用协议」最小代码（库 API）

```cpp
// 串口：选协议 0（三思）
comm.SetCommType(SERIAL);
comm.setParameter(QStringLiteral("iProtocolType"), 0);
comm.setParameter(QStringLiteral("iComPort"), 4);      // COM5（索引从 0）
comm.setParameter(QStringLiteral("iComBaud"), 5);      // 115200
comm.setParameter(QStringLiteral("iTriggerCtrl"), 1);  // 在 input 结构里，见 Demo setupSerial
// … input/output、波特等见 CommLab/src/DemoComm.h::setupSerial
bool ok = comm.Connect();

// 网口：选协议 2（中机）
comm.SetCommType(NETWORK);
comm.setParameter(QStringLiteral("iProtoType"), 2);
comm.setParameter(QStringLiteral("iTransferType"), 1); // UDP
comm.setParameter(QStringLiteral("sLocalIP"), QStringLiteral("127.0.0.1"));
comm.setParameter(QStringLiteral("iLocalPort"), 19105);
comm.setParameter(QStringLiteral("sDestIP"), QStringLiteral("127.0.0.1"));
comm.setParameter(QStringLiteral("iDestPort"), 19106);
ok = comm.Connect();
```

对照实现：`CommLab/src/DemoComm.h`。

## 3. 数据格式 / 组包 / 发送

### 3.1 两类载荷（必须分开）

| 类型 | 谁发 | 库信号 | 软件是否业务回发 |
|------|------|--------|------------------|
| 指令 | 试验机 | `emitEventMsg` | 否（可同步 `bOnlineCollect`） |
| 测数 | 试验机 | `emitNewData` | 是（若库有 `SendData` 且已开始/允许） |
| 业务结果 | 软件 | — | 库按协议组出站帧 |

### 3.2 组包在哪做

| 方向 | 谁组包 | 代码位置 |
|------|--------|----------|
| 机→软（入站） | **试验机 Demo** | `MachinePeer/src/DemoComm.h` 的 `packSerial*` / `packNet*` |
| 软→机（出站） | **库** | `SerialPortComm::SendData` / `SocketComm::SendData` |
| 软→机（解包） | **试验机 Demo** | `MachinePeer/src/ReplyParse.h` |

软件业务层只准备 `std::vector<double>`，**不要**自己拼协议字节（除非你在改库）。

### 3.3 软件发送业务结果（标准写法）

```cpp
// 前置：bInquireSendFlag == true（否则库 SendData 直接 return）
comm.setParameter(QStringLiteral("bInquireSendFlag"), true);

std::vector<double> v;
// 按协议填个数（审计）：
// 串口三思：通常 2 个（力、温）；科新：只需 1 个力；IEEE：≥5；冠腾：≥2
// 网口中机：恰好 2；纳百川线条：恰好 4；JSON：任意长度进 values
v.push_back(forceOut);
v.push_back(tempOut);

comm.SetCommType(SERIAL); // 或 NETWORK，与当前连接一致
comm.SendData(v, SERIAL);
```

对照：`CommLab/src/DemoComm.h::replyProcessed`。

### 3.4 试验机组测数帧（串口三思示例）

库收侧：HEX 串认 `E2` + 8 个 HEX 字符（4 字节大端力原始量）。

```cpp
// 力 1.0，量程约定与库 dInForceRange=100 时：raw ≈ force * 1e6
QByteArray packSansiForce(double force)
{
    const qint64 raw = qRound64(force * 1000000.0);
    const quint32 u = static_cast<quint32>(qBound(0LL, raw, 0xFFFFFFFFll));
    QByteArray frame;
    frame.append(char(0xE2));
    frame.append(char((u >> 24) & 0xFF));
    frame.append(char((u >> 16) & 0xFF));
    frame.append(char((u >> 8) & 0xFF));
    frame.append(char(u & 0xFF));
    return frame; // 例：1.0 → E2000F4240
}
```

对照：`MachinePeer/src/DemoComm.h::packSerialMeasure` case 0。

### 3.5 试验机发指令（串口三思）

```cpp
QByteArray start = QByteArray("\x3D\x0D", 2); // 开始
QByteArray stop  = QByteArray("\x3E\x0D", 2); // 停止

```

```cpp

QByteArray stop  = QByteArray("{\"tn\":3}");
```

## 4. 收包 / 解包 / 正确性 / 使用

### 4.1 软件怎么收（跟库信号，不要自己 read 串口）

```cpp
QObject::connect(&comm, &CommHandler::emitNewData, this,
    [](void* data, int size, int type) {
        // 注意：库线程可能直接回调 → 立即拷贝 double
        if (!data || size <= 0) return;
        QVector<double> values(size);
        memcpy(values.data(), data, size * sizeof(double));
        // 再 QueuedConnection 到 UI 使用 values
    }, Qt::DirectConnection);

QObject::connect(&comm, &CommHandler::emitEventMsg, this,
    [](int /*ctrl*/, int /*view*/, int msg) {
        // msg 为 W_CUSTOM_COMM_STARTCALC 等
    });
```

对照：`CommLab/src/CommController.cpp`（已做拷贝 + 排队）。

### 4.2 解包在哪发生

| 端 | 解包者 | 结果形态 |
|----|--------|----------|
| 软件收测数 | **库** | `double*` + size + type → 你的业务 |
| 软件收指令 | **库** | 事件码 |
| 试验机收业务回传 | **你的代码** | 按出站格式解析（`ReplyParse`） |

### 4.3 怎么判断解包正确

建议按层校验：

1. **帧界**：定长？结束符 `0x0D`？JSON 可 `fromJson`？  
2. **头/关键字**：如 `E2`、`024C`、`tn==2`、`msgType==0x04`。  
3. **长度**：`raw.size() == sizeof(ForceData)`。  
4. **字段语义**：三思/科新 `values[1]` 可能是**状态枚举**不是温度。  
5. **业务范围**：力温是否在合理区间。

### 4.4 解包后怎么用（软件）

```cpp
void onSafeData(QVector<double> values, int /*type*/)
{
    if (values.isEmpty()) return;
    double force = values.at(0);
    // 串口三思/科新：不要把 values[1] 当温度（状态位）
    double temp = 25.0;
    // … 你的算法
    double forceOut = force; // 示例：透传
    double tempOut = temp;
    // 条件：已连接、已开始、允许回发、协议有出站
    replyProcessed(handler, SERIAL, 0, forceOut, tempOut);
}
```

### 4.5 试验机解软件回包（科新：仅力）

库出站只有力 + `#`，**没有温字段** → 解析日志只打力，不要打印 `温=0`。


## 5. 正确性还能怎么判断与验证

| 方法 | 做法 | 适用 |
|------|------|------|
| 对照已知样例 | 力=1.0 → 三思 `E2000F4240` | 组包回归 |
| 双边日志 | Peer `TX` HEX ↔ Lab `RX F=` ↔ Peer `RX 解析` | 闭环 |
| 错包静默 | 故意发短包/错头，应无 `emitNewData` | 解析健壮性 |
| 指令/测数分离 | 只点开始，不应出 `TX F=` | 产品逻辑 |
| Wireshark / 虚拟串口监视 | 看真实字节 | 网口/串口 |
| 库测试场景 | `CommHandler/test/.../scenarios` | 中机 ForceData 等 |
| 校验和/CRC | 仅当协议定义了校验字段 | 自研协议推荐 |

**验证口诀**：能复现「发出的字节」与「对端日志」一一对应，才算通。

## 6. 联调常见问题（粘包、网络等）

### 6.1 串口

| 现象 | 原因 | 处理 |
|------|------|------|
| Lab 无任何 RX | COM 不成对 / 波特率不一致 / 未 Connect | 虚拟串口两端各占一口；参数一致 |
| Peer 有 TX，Lab 无 F= | 帧格式不对（如科新发了裸 float） | 按库 HEX 头重组成包 |
| Lab `T=1` 离谱 | 把状态枚举当温度 | 协议审计：第二通道语义 |
| 粘包 | 多次写入无结束符，或一次读到多帧 | 按结束符/`定长`缓冲拆帧（Peer 三思回包按 `0x0D`） |
| 半包 | 一帧分多次 `readyRead` | 缓冲直到长度够或遇结束符 |
| 冠腾能选但不能发力温 | 库无入站 case | 属能力限制，非 Demo bug |

### 6.2 网口

| 现象 | 原因 | 处理 |
|------|------|------|
| JSON 发 `tn=2` 力温 Lab 无反应 | 库收 JSON **不解析 values** | 换中机/威盛测数，或改库 |
| 开始后 Peer 收到 JSON ACK | 库自动应答 | 勿当业务回发 |
| 端口绑失败 | 端口占用 / 权限 | 换端口；两端 ini 成对 |
| UDP 丢包 | 无连接、无重传 | 联调本机环回；关键指令可重发 |
| TCP 粘包 | 流式边界不清 | 定长/长度前缀/分隔符 |
| 防火墙 | 拦 UDP | 放行本机端口 |
| 中机二进制当文本 | 日志乱码 | 用 HEX 打印 |

### 6.3 粘包 / 半包（通用处理模板）

```cpp
// 结束符协议：缓冲直到 0x0D
void onReadyRead()
{
    m_buf.append(port->readAll());
    int pos = 0;
    while ((pos = m_buf.indexOf(char(0x0D))) >= 0) {
        QByteArray frame = m_buf.left(pos + 1);
        m_buf.remove(0, pos + 1);
        handleFrame(frame); // 解一帧
    }
}

// 定长协议：缓冲直到 sizeof(Frame)
void onReadyReadFixed()
{
    m_buf.append(sock->readAll());
    while (m_buf.size() >= (int)sizeof(Frame)) {
        Frame f;
        memcpy(&f, m_buf.constData(), sizeof(f));
        m_buf.remove(0, sizeof(f));
        handleFrame(f);
    }
}
```

### 6.4 其它常见坑

- `bInquireSendFlag == false` → `SendData` 静默不发。  
- 未「开始」就期望回发 → Demo 用 `m_running` 门控。  
- 串口设 `bOnlineCollect` → **串口模块无此 key**，无效。  
- 网口/串口同时 Connect → 互斥，先断再开。  
- LNK1168 → exe 未关，无法覆盖编译。

## 7. 串口详解 + 可复制闭环示例

### 7.1 参数约定（与 Demo 一致）

| 项 | 软件 Lab | 试验机 Peer |
|----|----------|-------------|
| 通道 | 串口 | 串口 |
| 协议 | 0 三思 | 0 三思 |
| COM | 例如 COM4 | 配对口 COM5 |
| 波特 | 115200 8N1 | 同左 |

### 7.2 闭环步骤

1. Peer 打开串口 → Lab 连接。  
2. Peer **开始** → Lab `RX CMD 开始`。  
3. Peer **发力温** → Lab `RX F=` →（勾选回发）`TX F=` → Peer `RX` + `RX 解析`。  
4. Peer **停止** → Lab `RX CMD 停止`。

### 7.3 最小可粘贴：试验机发「开始 + 力」

```cpp
#include <QSerialPort>

bool openPort(QSerialPort& p, const QString& name)
{
    p.setPortName(name);
    p.setBaudRate(QSerialPort::Baud115200);
    p.setDataBits(QSerialPort::Data8);
    p.setParity(QSerialPort::NoParity);
    p.setStopBits(QSerialPort::OneStop);
    return p.open(QIODevice::ReadWrite);
}

QByteArray makeStart() { return QByteArray("\x3D\x0D", 2); }

QByteArray makeForce(double force)
{
    const quint32 u = static_cast<quint32>(qBound(0LL, qRound64(force * 1e6), 0xFFFFFFFFll));
    QByteArray f;
    f.append(char(0xE2));
    f.append(char((u >> 24) & 0xFF));
    f.append(char((u >> 16) & 0xFF));
    f.append(char((u >> 8) & 0xFF));
    f.append(char(u & 0xFF));
    return f;
}

// 使用：
// QSerialPort port; openPort(port, "COM5");
// port.write(makeStart());
// port.write(makeForce(1.0));
```

### 7.4 最小可粘贴：软件侧收测数并回发（概念）

完整工程请直接看 `CommLab`；逻辑浓缩如下：

```cpp
// Connect 后：
QObject::connect(ctrl, &CommController::eventReceived, [&](int, int, int msg) {
    if (msg == W_CUSTOM_COMM_STARTCALC) running = true;
    if (msg == W_CUSTOM_COMM_STOPCALC)  running = false;
});
QObject::connect(ctrl, &CommController::safeDataReceived, [&](QVector<double> v, int) {
    if (v.isEmpty() || !running) return;
    double f = v[0];
    // 处理…
    std::vector<double> out{f, 25.0};
    handler->setParameter(QStringLiteral("bInquireSendFlag"), true);
    handler->SendData(out, SERIAL);
});
```

### 7.5 科新注意（易错）

- 入站：`02 4C` + 9 字节 ASCII 力，不是裸 IEEE754。  
- 出站：只有力；Peer 解析不要报温。

## 8. 网口详解 + 可复制闭环示例

### 8.1 参数约定

| 项 | Lab（软件） | Peer（试验机） |
|----|-------------|----------------|
| 本机 | 127.0.0.1:19105 | 127.0.0.1:19106 |
| 对端 | 127.0.0.1:19106 | 127.0.0.1:19105 |

## 9. 实战：添加一个超级简单的自定义协议

下面用一个 **教学协议「MiniLine」**，刻意简单，展示完整增改路径。

### 9.1 协议规格（你自己定的规则）

| 项 | 约定 |
|----|------|
| 通道 | 先用 **纯 Qt 两端** 验证；再考虑是否进 CommHandler |
| 指令 | 文本行：`C:START\n` / `C:STOP\n` |
| 测数 | 文本行：`D:<force>,<temp>\n` 例：`D:1.000,25.000\n` |
| 回发 | 文本行：`R:<force>,<temp>\n` |
| 校验 | 行以 `\n` 结束；`sscanf`/`split` 成功；力温有限数值 |

> **严谨说明**：若软件必须走现有 `CommHandler.dll`，必须在库 `SerialPortComm`/`SocketComm` 增加新 `case`。  
> 下面先给 **Peer ↔ 最小软件伪代码** 全链路；再列 **进库改动清单**。

### 9.2 组包 / 解包（可复制）

```cpp
// MiniLinePack.h — 极简文本行协议
#pragma once
#include <QByteArray>
#include <QString>
#include <cmath>

// 组开始指令
inline QByteArray miniPackStart() { return QByteArray("C:START\n"); }
// 组停止指令
inline QByteArray miniPackStop()  { return QByteArray("C:STOP\n"); }

// 组测数
inline QByteArray miniPackData(double force, double temp)
{
    return QStringLiteral("D:%1,%2\n")
        .arg(force, 0, 'f', 3)
        .arg(temp, 0, 'f', 3)
        .toLatin1();
}

// 组业务回发
inline QByteArray miniPackReply(double force, double temp)
{
    return QStringLiteral("R:%1,%2\n")
        .arg(force, 0, 'f', 3)
        .arg(temp, 0, 'f', 3)
        .toLatin1();
}

struct MiniParsed {
    bool ok = false;
    bool isCmd = false;
    bool isData = false;
    bool isReply = false;
    bool isStart = false;
    double force = 0;
    double temp = 0;
};

// 解一行（已去掉或保留末尾 \n 均可）
inline MiniParsed miniParseLine(QByteArray line)
{
    MiniParsed r;
    if (line.endsWith('\n')) line.chop(1);
    const QString s = QString::fromLatin1(line).trimmed();
    if (s.startsWith(QLatin1String("C:"))) {
        r.ok = true;
        r.isCmd = true;
        r.isStart = (s == QLatin1String("C:START"));
        return r;
    }
    if (s.startsWith(QLatin1String("D:")) || s.startsWith(QLatin1String("R:"))) {
        const QString body = s.mid(2);
        const QStringList p = body.split(QLatin1Char(','));
        if (p.size() != 2) return r;
        bool okF = false, okT = false;
        r.force = p[0].toDouble(&okF);
        r.temp = p[1].toDouble(&okT);
        if (!okF || !okT || !std::isfinite(r.force) || !std::isfinite(r.temp))
            return r;
        r.ok = true;
        r.isData = s.startsWith(QLatin1String("D:"));
        r.isReply = s.startsWith(QLatin1String("R:"));
        return r;
    }
    return r;
}
```

### 9.3 粘包拆行接收

```cpp
QByteArray rxBuf;

void onBytes(const QByteArray& chunk)
{
    rxBuf.append(chunk);
    int n = 0;
    while ((n = rxBuf.indexOf('\n')) >= 0) {
        QByteArray one = rxBuf.left(n + 1);
        rxBuf.remove(0, n + 1);
        MiniParsed p = miniParseLine(one);
        if (!p.ok) { /* 记 ERR 行 */ continue; }
        if (p.isCmd) { /* 开始/停止 */ }
        if (p.isData) { /* 处理力温，再 write(miniPackReply(...)) */ }
        if (p.isReply) { /* 试验机显示算好的力温 */ }
    }
}
```

### 9.4 完整闭环（逻辑顺序）

```text
Peer: write(C:START\n)
Soft: 置 running=true
Peer: write(D:1.000,25.000\n)
Soft: 解 D → 处理 → write(R:1.000,25.000\n)
Peer: 解 R → 显示力温
Peer: write(C:STOP\n)
```
