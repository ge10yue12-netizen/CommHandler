# Demo_test 联调测试手册

> **开发自学（协议增删、组包解包、粘包排错、自定义协议）**：见同目录 [`通信协议开发完整手册.md`](通信协议开发完整手册.md)。  
> 角色：**CommLab** = 软件（调 `CommHandler`）；**MachinePeer** = 试验机工具（纯 Qt，不调 DLL）。  
> 原则：试验机发指令/测数 → 库解析 → 软件跟库信号做事；**库没有的路径不测、不当缺陷**。

---

## 1. 准备

| 项 | 说明 |
|----|------|
| 编译 | `CommLab` / `MachinePeer` 均为 x64 Debug（或 Release） |
| 串口 | 需虚拟串口对（如 COM5↔COM6）；Lab `port_index` 与 Peer 各占一端 |
| 网口 | 默认 Lab `127.0.0.1:19105`，Peer `127.0.0.1:19106`（见 ini） |
| 互斥 | **同一会话只开串口或只开网口**，不要双开 |
| 勾选 | Lab「收测后回发」默认勾选（业务回发需要） |

**推荐顺序：** 先串口三思（闭环最完整）→ 再网口中机 → 再其余协议按能力表点验。

## 2. 能力速查（测什么 / 不测什么）

### 2.1 网口 `iProtoType`

| PT | 名称 | 测指令 | 测测数 RX | 测业务回发 TX |
|----|------|--------|-----------|---------------|
| 0 | JSON | ✓ 开始/停止 | ✗ 库不解析 values | ✗ 无测数则无回发* |
| 1 | 万测 | ✓ START/STOP | ✗ | ✗（无测数） |
| 2 | 中机 | ✓ 二进制控制 | ✓ ForceData | ✓（需已开始+回发勾选） |
| 3 | 三思 | ✗ | ✓ 定长力 | ✗ 库无出站 |
| 4 | 触发存图 | ✓ 存图载荷 | ✗ | ✗ |
| 5 | 福建威盛 | ✗ | ✓ 文本力 | ✗ 库无出站 |
| 6 | 纳百川 | ✓ alphaStart/Stop | ✗ | 可另测 SendData 路径 |
| 7 | 纳百川线条 | ✓ calcStart/End | ✗ | 可另测（需 4 个数） |

\* JSON 下库对 `tn=1` 的自动 ACK ≠ 业务力温回发；Peer「发力温」应禁用或报 ERR。

### 2.2 串口 `iProtocolType`

| PT | 名称 | 测指令 | 测测数 RX | 测业务回发 |
|----|------|--------|-----------|------------|
| 0 | 三思 | ✓ 3D/3E/6B/5A | ✓ E2 力（第二通道为状态非温） | ✓ |
| 1 | 科新 | ✗ | ✓ 024C+ASCII 力（第二通道为状态非温） | ✓ **仅力**（无温字段） |
| 2 | 时代新材 | ✗ | ✓ CSV+RUN | ✗ 库 SendData 空 |
| 3 | IEEE | ✓ 2400/24FF（**库有**） | ✗ 收路径不解析测数 | 有出站；无入站则 Demo 不发测数 |
| 4 | 冠腾 | ✗ | ✗ | 有出站 R#N#；**库无入站**，Peer 只解析软件回包 |

## 3. 通用步骤

### 3.1 网口会话

1. Peer：通道=网口，选协议，本机 `19106`，软件 `19105` → **打开**  
2. Lab：通道=网口，**同一协议**，本机 `19105`，对端 `19106` → **连接**  
3. 状态：Peer「已打开」、Lab「已连接」  
4. 按该协议能力点 **开始 / 停止 / 发力温**（灰掉的按钮表示库无此能力）

### 3.2 串口会话

1. Peer / Lab 通道均=串口，**同一协议**，COM 号成对  
2. Peer **打开** → Lab **连接**  
3. 按能力表测指令与「发力温」

### 3.3 日志约定

| 前缀 | 含义 |
|------|------|
| `TX` | 本端发出 |
| `RX` | 本端收到原始报文（HEX 或文本） |
| `RX ACK` | 库/协议自动应答（非算好的力温） |
| `RX 解析 力= 温=` | **试验机已按协议解出软件回传的业务结果** |
| `RX CMD` | Lab 收到库 `emitEventMsg`（开始/停止等） |
| `RX F=` | Lab 收到库 `emitNewData` |
| `TX F=` | Lab 业务回发（`SendData(vector)`） |
| `ERR` | 本协议库无此能力或组包失败 |
| `INFO` | Lab 已收测数但库无业务回发 |

软件回发闭环核对：Lab 出现 `TX F=` 后，Peer 应出现对应的 `RX 解析 力=… 温=…`（库无出站的协议除外）。

## 4. 必测用例（建议全部勾选执行）

### A. 串口三思（完整闭环）— **P0**

| 步骤 | 操作 | 期望 |
|------|------|------|
| A1 | Peer 开始 | Lab `RX CMD 开始`，状态「运行中」 |
| A2 | Peer 发力温（如 1.0 / 25） | Lab `RX F=`；若勾选回发则 `TX F=`；Peer `RX` + **`RX 解析 力=`** |
| A3 | Peer 停止 | Lab `RX CMD 停止` |
| A4 | 未开始就发力温 | Lab 可有 `RX F=`，但不应业务回发（或无 `TX F=`） |
| A5 | 存图 / 清零 | Lab 仅 `RX CMD 存图/清零`（无相机实现即可） |

### B. 网口 JSON（仅指令）— **P0**

| 步骤 | 操作 | 期望 |
|------|------|------|
| B1 | Peer 开始 | Lab `RX CMD 开始`；Peer 可能收到库自动 JSON ACK |
| B2 | Peer 停止 | Lab `RX CMD 停止` |
| B3 | 「发力温」 | 按钮禁用或 `ERR`（说明库不解析测数）— **不算软件 bug** |

### C. 网口中机（测数+回发）— **P0**

| 步骤 | 操作 | 期望 |
|------|------|------|
| C1 | Peer 开始 | Lab `RX CMD 开始`；Peer 可有二进制 ACK（HEX） |
| C2 | Peer 发力温 | Lab `RX F=`；勾选回发则 `TX F=`；Peer **`RX 解析 力=`** |
| C3 | Peer 停止 | Lab `RX CMD 停止` |

### D. 网口三思 / 威盛（只收力）— **P1**

| 步骤 | 操作 | 期望 |
|------|------|------|
| D1 | 选协议 3 或 5，发力温 | Lab `RX F=`；`INFO` 无业务回发；指令按钮按能力灰掉 |

### E. 网口万测 / 纳百川（只指令）— **P1**

| 步骤 | 操作 | 期望 |
|------|------|------|
| E1 | 万测：开始/停止 | Lab `RX CMD`；发力温应 ERR |
| E2 | 纳百川 6：开始/停止 | Lab `RX CMD`；发力温应 ERR |
| E3 | 纳百川 7：calcStart/calcEnd | 同上 |

### F. 网口触发存图 — **P2**

| 步骤 | 操作 | 期望 |
|------|------|------|
| F1 | Peer 存图 | Lab `RX CMD 触发存图`；开始/发力温不可用 |

### G. 串口科新 / 时代新材 — **P1**

| 步骤 | 操作 | 期望 |
|------|------|------|
| G1 | 科新：打开后直接发力温 | Peer `TX` 含 `024C`；Lab `RX F=`（无伪 T=1）；`TX F=`（仅力）；Peer `RX 解析 力=`（不出现温=0） |
| G2 | 时代新材：发力温 | Lab `RX F=`；`INFO` 无业务回发 |

### H. 串口 IEEE — **P2**

| 步骤 | 操作 | 期望 |
|------|------|------|
| H1 | 开始/停止 | Lab `RX CMD` |
| H2 | 发力温 | ERR（库不解析测数） |

### I. 通道互斥 — **P0**

| 步骤 | 操作 | 期望 |
|------|------|------|
| I1 | Lab 已连串口时切到网口再连 | 须先断开；连接时只生效当前通道 |

## 5. 常见现象说明（勿判为缺陷）

| 现象 | 原因 |
|------|------|
| JSON 发了力温 Lab 无 `RX F=` | 库 `ParsingProtocolI` 不读 `values` |
| Peer 开始后收到 JSON `code:0` | 库自动 ACK，不是软件业务回发 |
| 三思/威盛网口有 `RX F=` 无 `TX F=` | 库无该协议 `SendData(vector)` |
| 中机/纳百川开始后库已置采集标志 | 软件仍同步 UI「运行中」即可 |
| 三思回发出现 `T=25` 而 Peer 填了 30 | **已修**：E2 入站无温，禁止用缺省 25 回发；应只见 `TX F=` |
| `processForceTemp` | Demo 为**透传占位**（可换成真实标定）；不是未调用 SendData |
